# v0.8 Alpine.js + HTMX trade-off benchmark

Honest accounting of what the v0.8 frontend stack costs vs saves.
Generated 2026-05-13 against the current main (commit `b8a70a2`).

## 1. Current inline JS in templates

Measured via `re.findall(r'<script[^>]*>(.*?)</script>', ...)` +
inline `onclick=`/`onchange=` handlers. Bytes are **uncompressed
source** (the gzip-on-the-wire cost is ~30-40% of this).

| Template          | inline JS (B) |
|---|---:|
| base.html         |  2,336 |
| detail.html       |    758 |
| docs_page.html    |    997 |
| list.html         |  2,094 |
| multi_track.html  |    963 |
| **qa.html**       | **40,785** |
| **TOTAL**         | **47,933** |

The `qa.html` block dominates everything else by a factor of 19×.
It carries: localStorage state per checkbox (73 items), progress-bar
update, full EN/FR i18n switcher (121 keys × 2 langs hard-coded
in the JS), reset confirm dialog, per-section toggle handlers.

Net per-pageload **uncompressed** bytes:
- Every page → 2,336 B (`base.html` theme-toggle + transition logic)
- `/_qa/` page → +40,785 B
- Others → +700–2,100 B each

**Gzipped equivalent** (the wire cost): roughly ~30–40% of source.
So `qa.html` alone ships ≈ 13 KB over the wire today.

## 2. Library cost (CDN, gzipped)

| Library         | gzipped wire |
|---|---:|
| Alpine.js 3.14  | ≈ 15 KB |
| HTMX 2.0.4      | ≈ 14 KB |
| **Total**       | **≈ 29 KB** |

Both are loaded **once**, cached by the browser, shared across every
page. So the 29 KB is amortised across all viewer pageloads in a
session — not paid per page.

## 3. The actual trade-off, per page

| Scenario                                            | First page | Nth page |
|---|---:|---:|
| Today (inline JS, no libs)                          | inline only | inline only |
| With libs, NO refactor (current `b8a70a2` state)    | inline + 29 KB libs | inline (libs cached) |
| With libs + full Alpine/HTMX refactor (target)      | ~25% of inline + 29 KB | ~25% of inline |

**Specifically for `qa.html`**:
- Today: ~13 KB on the wire (only-when-`--debug`).
- With Alpine: estimated ~3–5 KB inline (`x-data` directives on the
  checkboxes, no manual addEventListener bookkeeping) + the 29 KB
  libs amortised. Net: −8 to −10 KB on `qa.html` itself, +29 KB
  one-time across the session.

**Break-even**: ≈ 3 pageloads of `qa.html` per session for the
Alpine refactor to pay off vs vanilla. The QA workflow easily
crosses that (people open the checklist + scroll + tick).

**For non-debug users** (no `qa.html` access): the libs don't
amortise. The 29 KB is pure overhead unless we use HTMX for
record-list pagination + sidebar partial swaps (which is a separate
v0.8.2 PRD's scope).

## 4. Decision matrix

| Path | When | Cost | Verdict |
|---|---|---|---|
| **Keep status quo** (libs loaded, no refactor) | now | +29 KB always | **worst** — pay both |
| **Drop the libs entirely** | now | 0 KB but slow HTMX adoption | viable; reverts the 1 HTMX form to GET reload |
| **Full Alpine refactor of `qa.html`** | follow-up | -10 KB on qa, +29 KB session-wide | net win for `--debug` users only |
| **Full Alpine refactor everywhere + 3+ HTMX surfaces** | v0.8.2 | -25 KB inline, +29 KB session | net win for everyone after 2-3 pageloads |

## 5. Honest recommendation (from this benchmark)

The current state (`b8a70a2`) is the worst of both worlds: libs
loaded but only one form actually uses them.

Two viable next steps:

**Option A — Revert libs, keep HTMX form**: drop the CDN script tags
from `base.html`, downgrade the multi-track form to a plain GET
reload. Cost: lose the partial-swap UX on one form. Saves: 29 KB on
every pageload. **Right move if v0.8.2 (vendoring + adoption) is
deferred indefinitely.**

**Option B — Commit to the refactor**: spend the budget on porting
`qa.html` + the 5 other inline-JS surfaces to Alpine/HTMX. Then the
libs amortise. **Right move if v0.8.2 will ship within the next 1-2
release cycles.**

The PR landing this benchmark is the trigger to pick a side. Either
path is honest; the half-finished current state is not.

## 6. How to reproduce

```bash
.venv/bin/python -c "
import re
from pathlib import Path
total = 0
for tpl in sorted(Path('ulog/web/templates/ulog').glob('*.html')):
    html = tpl.read_text()
    n = sum(len(m.group(1)) for m in re.finditer(
        r'<script(?![^>]*\ssrc=)[^>]*>(.*?)</script>', html, re.DOTALL))
    n += sum(len(m.group(1)) for m in re.finditer(
        r'on(?:click|change|input|submit)=\"([^\"]+)\"', html))
    if n: print(f'{tpl.name:<32} {n:>10,}')
    total += n
print(f'{\"TOTAL\":<32} {total:>10,}')
"
```
